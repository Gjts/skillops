# v0.4 UX progress

> Updated: 2026-07-23
> Status: cost provenance and Runs pagination P0 correction implemented; the broader v0.4 UX task remains in progress

## Completed correction

- Runtime cost is reported only when a terminal Skill run contains a finite
  `costUsd`. Missing metadata renders as **Not reported**; an explicit zero
  renders as `$0.00`.
- Runtime Skill run cost remains separate from Promptfoo and Provider Evaluation
  cost.
- `GET /api/runs` validates page, page size, query, runtime, project, outcome,
  date, sort, and cost filters; it returns one stable terminal-run page and
  scoped lifecycle counts from the existing JSONL event store.
- Canonical `GET /api/runs/~:id` returns one run in a bounded 200-event
  correlation window that preserves the run and reports total/truncation metadata.
- Runs supports first/previous/neighbor/next/last controls, 20/50/100 rows per
  page, URL-restorable state, success-only history entries, request
  cancellation/sequence protection, full-state failure rollback, deletion-aware
  page clamping, identity-aware bounded new-record polling without `/api/events`,
  and a local seeded fallback whose Runs transport retries the API and exits
  Demo mode after recovery without relabeling the separate full event feed.
- Run detail uses a dot-safe canonical route, mirrors runtime/session/turn
  correlation on both sides, and labels bounded-window truncation with loaded/total counts.
- Legacy JSONL rows with missing or malformed IDs receive deterministic,
  duplicate-aware IDs that remain stable through rewrites and explicit migration.
